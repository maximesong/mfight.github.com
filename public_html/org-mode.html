安装

激活

在配置文件中增加以下配置

;; the following lines are always needeed. Choose your own keys
(add-to-list 'auto-mode-alist '("\\.org\\'" . org-mode))
;; not needed when global-font-lock is on
(add-hook 'org-mode-hook 'turn-on-font-lock)
(global-set-key "\C-cl" 'org-store-line)
(global-set-key "\C-ca" 'org-agenda)
(global-set-key "\C=cb" 'org-iswitchb)

使用

标题和纲要

org-mode采用层次式的结构来安排文档，这是一种记录笔记和思想的的好方法。通过折叠大部分的内容，你可以通过那些展开的内容来获得全局的概要，和你关心和使用的部分。

层次结构的标题

通过行首一个或多个*来给出标题的层次化结构，如：

* Top level headline
** Second level
*** 3rd level
    some Text
*** 3rd level
    more text

内容的展开与折叠循环

纲要的使用使得隐藏部分的文本成为可能。org-mode使用两个命令来进行折叠和展开内容的循环，它们被分别绑定到TAB和S-TAB(或是"C-u TAB")。

TAB 子树循环：改变当前子树的状态

—> FOLDER —> CHILDREN —> SUBTREE —

S-TAB 和 C-u TAB 全局循环：改变整个Buffer的状态

—> OVERVIEW —> CONTENTS —> SHOW ALL —

全局状态

当Emacs第一次打开一个org文件的时候，全局状态被设为OVERVIEW，也就是只有顶层的标题是可见的。这个默认设定可以通过配置变量org-startup-folded来进行改变。对于每一个文件，也可以通过增加一个启动的关键词来overview, content, showall进行配置，比如：

#+STARTUP: content

移动

C-c C-n

下一个标题

C-c C-p

上一个标题

C-c C-f

下一个同层标题

C-c C-b

上一个同层标题

C-c C-u

回到上一层标题

结构修改

M-RET

插入一个和当前同层的标题。如果游标在一个普通列表项中，那么会插入一个新的列表项。

M-S-RET

插入一个新的TODO项，和当前的标题同层

M-LEFT/RIGHT

增加或减少当前标题的层数

M-S-LEFT/RIGHT

增加或减少当前子树的层数

M-S-UP/DOWN

向上/下移动子树。

C-c C-w

把一段内容定位到另一个标题中

C-x n s/w

把Buffer内容限定到当前子树/恢复到全部内容

稠树（Sparse trees)

C-c /

通过用户输入，构建一个sparse tree。

C-c / r

通过输入正则表达式来显示所有匹配的稠树。每个匹配都是高亮的，可以通过"C-c C-c "取消高亮。

普通列表

无序列表

以-、+、*作为开头。

有序列表

以1., 1)作为开头。

描述列表

用::来分开术语和描述

同一个列表的列表项必须在第一行使用相同的缩进。在一个列表项的下一行使用和它一样的标记和缩进时就结束了。一个列表在所有列表项结束，或者有两行空白行的时候结束。

快捷键
TAB

折叠项，就和列表层次一样。

M-RET

在当前层次插入一个项。

M-S-RET

插入一个具有勾选框的项

M-LEFT/RIGHT

增加或减少项的缩进，子项并不变动。

M-S-LEFT/RIGHT

增加或减少项的缩进，连同子项。

C-c C-c

如果在项的这一行有勾选框，那么改变勾选框的状态。

C-c -

在整个列表层循环。

脚注

快捷键

脚注的定义以一个在方括号内的标记开始。定义必须从行首开始，不能有缩进。脚注的引用就是简单地在文本里加入带有方括号的标记，比如：

The Org homepage[fn:1] now looks a lot better than it used to.
...
[fn:1] The link is: http://orgmode.org
C-c C-x f

当光标在一个脚注的引用上时，跳到定义处。如果在定义处，就跳到第一个引用。否则的话，就新建一个脚注。

C-c C-c

在定义和引用间跳转
